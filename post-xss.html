Cross-Site Scripting (XSS)

XSS permite a un atacante inyectar código JavaScript en una página web que otros usuarios visitan, provocando robo de datos, suplantación de identidad o acciones automatizadas.

Tipos de XSS

  • Reflejado: El payload se refleja directamente en la respuesta.
  • Persistente: El script se guarda en la base de datos y se ejecuta cada vez que se carga la página.
  • DOM-based: Se ejecuta debido a manipulaciones del DOM en el lado del cliente.

Ejemplo de XSS reflejado

URL: https://example.com?q=<script>alert('XSS')</script>

Respuesta del servidor:
Hola <script>alert('XSS')</script>

Cómo se explota

  • Campos de búsqueda, formularios, URLs mal sanitizados
  • Parámetros que se insertan directamente en el DOM
  • Contenido HTML generado dinámicamente sin escaping

Prevención

1. Escapa todo el contenido dinámico en HTML: 
<%= userInput %> → correcto
<%= raw userInput %> → ❌ vulnerable
2. Usa CSP (Content Security Policy): 
Content-Security-Policy: default-src 'self'; script-src 'self'
3. Valida y filtra entrada del usuario: 
re.match("^[a-zA-Z0-9 ]+$", entrada)
4. Usa frameworks que hacen escaping automáticamente:
  • React
  • Angular
  • Vue (con delimitadores estándar)

Consecuencias de XSS

  • Robo de cookies y sesiones
  • Phishing y suplantación de interfaces
  • Control total del navegador víctima

Conclusión

XSS es tan peligroso como común. Escapa el contenido, valida siempre lo que venga del usuario y evita crear HTML desde strings sin filtros.
